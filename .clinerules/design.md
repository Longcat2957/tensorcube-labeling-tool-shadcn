## 0. 핵심 디자인 원칙 (Design Principles)

1. **shadcn-svelte 우선 (Shadcn-First)**: UI의 일관성과 개발 속도를 위해 커스텀 CSS를 최소화하고, 모든 기본 UI 요소는 `shadcn-svelte`의 컴포넌트(Button, Slider, Tabs, ScrollArea, RadioGroup 등)를 확장하여 조립한다.
2. **단일 책임 및 재사용성 (Reusability)**: Navigation, Sidebar, Tools 등은 독립적인 Svelte 컴포넌트로 분리(`.svelte`)하여 다른 라벨링 타입(BB, OBB 등)에서도 props만 변경하여 재사용할 수 있도록 설계한다.
3. **100% 단축키 지원 (Keyboard-Driven)**: 마우스 이동 동선을 최소화하기 위해 **모든 버튼, 툴 스위칭, 탭 전환에 단축키가 매핑**되며, UI 툴팁(`Tooltip` 컴포넌트)을 통해 사용자에게 단축키를 명시적으로 노출한다.

## 1. 전체 레이아웃 구성 (Layout Architecture)

화면은 전체적으로 **세로 3분할(Top - Mid - Bottom)** 구조를 가지며, 가장 넓은 영역인 Main은 다시 **가로 3분할(Left - Center - Right)** 로 구성된다. CSS Grid 또는 Flexbox를 사용하여 창 크기 조절 시 유연하게 반응(Responsive)하도록 설계한다.

```
+-------------------------------------------------------------+
| [1.1 Navigation]  (Menu) WorkspaceName | [Tabs] [Export UI] |
+------+-----------------------------------------------+------+
| [1.2 |                                               | [1.4 |
| Left |                                               | Right|
| Side-|               [1.3 Canvas Area]               | Side-|
| bar] |                                               | bar] |
|      |                                               |      |
| [T]  |               (Image / Labels)                | [M]  |
| [o]  |                                               | [i]  |
| [o]  |                                               | [n]  |
| [l]  |                                               | [i]  |
| [s]  |                                               | [m]  |
|      |                                               | [a]  |
|      |                                               | [p]  |
+------+-----------------------------------------------+------+
| [1.5 Footer / Status Bar]   X: 1920, Y: 1080 | Zoom: 150%   |
+-------------------------------------------------------------+
```

### 1.1 Navigation (상단 바)

작업 공간의 전반적인 메타 기능을 제어한다. 높이는 고정(예: `h-14`)하여 작업 공간을 침범하지 않도록 한다.

- **좌측 영역 (메뉴 및 정보)**
  - **Menu Dropdown**: `DropdownMenu` 컴포넌트를 사용한 햄버거 버튼. (프로젝트 설정, 다른 워크스페이스 열기, 종료 등)
  - **Workspace Name**: 현재 작업 중인 프로젝트의 이름 (`workspace.yaml`에서 로드).
- **중앙/우측 영역 (작업 모드 전환 및 내보내기)**
  - **Mode Tabs**: `Tabs` 컴포넌트를 사용하여 기능 전환.
    - `Edit (작업)` | `Check (검수)` | `Preview (미리보기)`
    - 단축키: `Tab` 또는 `Shift+Tab`으로 모드 순환.
  - **Export Button**: 가장 우측에 위치. `Button` 컴포넌트(`variant="outline"`, `size="icon"`)에 내보내기 아이콘 구동. 클릭 시 데이터셋 추출 모달(`Dialog`) 오픈.

### 1.2 Main - Left Side Bar (좌측 툴바)

라벨링 툴의 조작 도구들이 세로(Flex-col)로 일렬 배치되는 영역. 작업(과업) 타입에 따라 렌더링되는 아이콘 리스트가 동적으로 변경된다.

- **디자인 및 구현**: 모든 버튼은 `shadcn`의 `Button(size="icon", variant="ghost")`를 사용하며, 활성화된 도구는 `variant="secondary"` 등으로 하이라이팅 처리한다. 모든 버튼은 `Tooltip`으로 감싸져 마우스 오버 시 단축키를 표시한다.
- **구성 요소 (예시)**:
  - ⬅️ 이전 이미지 이동 (단축키: `A`)
  - ➡️ 다음 이미지 이동 (단축키: `D`)
  - 🖱️ 선택/이동 툴 (단축키: `V`)
  - 🔲 바운딩 박스 생성 툴 (단축키: `B` - 과업 타입에 따라 일반 박스 또는 회전 박스로 작동)
  - ↩️ 실행 취소 (Undo) (단축키: `Ctrl + Z`)
  - ↪️ 다시 실행 (Redo) (단축키: `Ctrl + Y` 또는 `Ctrl + Shift + Z`)
  - 🗑️ 선택된 라벨 삭제 (단축키: `Delete`)

### 1.3 Main - Canvas (중앙 캔버스 및 뷰어)

이미지와 데이터가 렌더링되는 핵심 영역. Svelte의 생명주기와 HTML5 `<canvas>` (또는 Fabric.js 등)를 결합하여 구현한다.

- **뷰어 컨트롤 플로팅 바 (Floating Bar)**: 캔버스 상단이나 하단에 반투명한 패널로 배치.
  - **라벨 숨기기/보기 Toggle**: `Switch` 컴포넌트. (단축키: `H`). 활성화 시 원천데이터(원본 이미지)만 깔끔하게 확인 가능.
  - **라벨 불투명도 조절**: `Slider` 컴포넌트. 라벨 박스와 내부 채우기(Fill)의 투명도를 0% ~ 100%로 조절하여 가려진 객체를 확인.
- **인터랙션 경험 (UX)**:
  - **그리기**: 우측 패널(Right Side Bar)에서 선택된 '클래스'의 색상과 ID를 상속받아, 캔버스에서 마우스를 Click & Drag 하면 즉시 해당 클래스의 박스가 생성된다.
  - **상태 시각화**: 선택된 박스는 모서리에 Resize 핸들(Anchor)이 나타나며 선의 두께가 두꺼워진다.

### 1.4 Main - Right Side Bar (우측 인스펙터 패널)

전체 구조와 레이어 층위를 시각적으로 제공하는 영역. 세로 스크롤이 발생할 수 있으므로 전체를 `ScrollArea` 컴포넌트로 감싼다.

- **(상단) Minimap (전체 미리보기)**
  - 전체 이미지의 썸네일을 띄우고, 캔버스가 확대(Zoom-in)되면 현재 보고 있는 시야(Viewport) 영역을 빨간색 네모 프레임으로 표시한다.
  - 이 빨간 프레임을 미니맵 안에서 드래그하여 캔버스의 시점(Panning)을 빠르게 이동할 수 있다.
- **(중단) Class Selector (클래스 라디오 리스트)**
  - `workspace.yaml`에서 불러온 클래스 리스트가 표시된다.
  - `RadioGroup` 형태를 사용하여 **반드시 하나의 클래스가 활성화(Selected)** 되어 있도록 강제한다. (이 상태가 캔버스 드래그 시 생성될 박스의 클래스를 결정함)
  - **단축키 연동**: 각 클래스 좌측에 번호 배치를 렌더링하고 숫자 키보드(`1`, `2`, `3` ...)를 누르면 즉시 해당 클래스로 스위칭된다.
- **(하단) Label Layer List (라벨링 리스트)**
  - 포토샵의 레이어 패널과 유사하게 작동. 현재 이미지에 생성된 라벨 목록이 리스트 형태(`Card` 또는 촘촘한 리스트)로 표시된다.
  - 특정 라벨 항목을 클릭하면 캔버스 상표에서도 해당 박스가 '선택된 상태'로 하이라이트 된다. 역으로 캔버스에서 박스를 클릭해도 리스트에서 해당 줄이 스크롤링되며 하이라이트 된다.
  - 각 리스트 아이템에는 개별 [눈알 아이콘(숨기기)], [휴지통 아이콘(삭제)] 이 배치된다.

### 1.5 Footer (하단 디버그 및 상태 바)

사용자의 작업 상태와 미세한 데이터를 실시간으로 보여주는 슬림한 바(Bar). (`text-xs`, `text-muted-foreground` 색상 활용)

- **좌측**: 현재 시스템 상태 메시지 (예: "자동 저장됨 (10:35)", "데이터를 로딩 중입니다...")
- **중앙**: 현재 이미지 파일명 및 진척도 (예: `0000000002_W.json [ 2 / 5000 ]`)
- **우측 디버그 정보**:
  - 마우스의 캔버스 내 절대 좌푯값 (`X: 1450, Y: 920`)
  - 현재 이미지 확대 배율 (`Zoom: 150%`)

## 2. 컴포넌트 데이터 흐름 및 상태 관리 전략 (Svelte 중심)

본 프로젝트는 재사용성을 극대화하기 위해 Svelte의 **Context API를 주로 사용한다. 다음의 원칙을 준수할 것**

1. +page.svelte →자식 컴포넌트간 값의 참조 및 갱신은 반드시 Context API를 사용한다.
   - 전역으로 참조되는 변수는 반드시 +page.svelte 내부에 setContext형식으로 정의가 되어야 한다는 거다.
2. 자식 컴포넌트 → 자식 컴포넌트간 값의 참조 및 갱신은 기존과 같이 `$props`, `$bindable`을 사용한다.
