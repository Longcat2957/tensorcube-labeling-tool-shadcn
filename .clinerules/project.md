## 1. 프로젝트의 목적

본 프로젝트는 `Svelte + Electron`으로 구성되었으며 Windows, Mac, Linux 등 다양한 OS에 대응 가능한 단독 프로그램(Standalone)형 데이터 라벨링 툴이다. 이를 통해 다양한 환경에서 안정적으로 구동할 수 있을 뿐만 아니라, 추후 AI 추론(Inference) 기능을 연동하여 라벨링의 편의성과 속도를 극대화하는 스마트 작업 환경 구축을 목표로 한다.

### 1.1 프로젝트에서 지원하는 라벨링 방식

1. **Bounding Box (BB)**: 축 정렬(Axis-aligned) 바운딩 박스. (타입 `1`형)
2. **Oriented Bounding Box (OBB)**: 회전 바운딩 박스. 객체의 방향성 및 각도를 포함. (타입 `2`형)
*(※ 추후 Polygon, Keypoint 등 추가 형태의 라벨링으로 확장 가능성을 열어둔다.)*

## 2. 프로젝트에 포함된 의존성

프로젝트에 사용되는 대표적인 기술 스택 및 의존성은 다음과 같다.

- **Svelte**: 빠르고 반응성 높은 프론트엔드 UI/상태 관리
- **Electron**: 크로스 플랫폼 데스크톱 애플리케이션 빌드 및 로컬 파일 시스템 제어
- **shadcn-svelte (w. tailwindcss v4)**: 직관적이고 일관된 디자인 시스템 및 UI 컴포넌트 구성
- *(추후 도입 예정)* **TensorFlow.js / ONNX Runtime Web**: 로컬 환경에서의 AI 모델 구동 (Auto-labeling 등)

## 3. 용어 정리

- **workspace (워크스페이스)**: 개별 워크스페이스는 각각 독립적인 데이터 라벨링 과업을 의미한다. `workspace.yaml`을 읽는 것으로 이미 생성된 워크스페이스를 불러오기(Load) 할 수 있다.
- **원천데이터 (Source Data)**: 라벨링의 대상이 되는 원본 이미지 파일을 의미한다.
- **라벨데이터 (Label Data)**: 유저가 작업한 라벨링 결과물. 라벨데이터는 반드시 정규화되지 않은 **pixel-by-pixel 형식의 절대 좌표(실수 값)** 로 저장된다. 소수점 좌표를 그대로 저장하여 정밀한 위치 표현이 가능하며, 내보내기(Export) 시 필요에 따라 정수로 변환된다.
- **내보내기 (Export)**: 로컬에 저장된 pixel 절대 좌표 데이터를 YOLO, COCO, DOTA(OBB용) 등 AI 모델 학습에 맞는 일반적인 규격으로 변환 추출하는 과정. 내보내기를 통해 원천데이터의 유효성을 다시 한번 확인하며, 리사이즈 기능을 통해 데이터의 해상도/용량을 컨트롤하고, Train/Val/Test 등 데이터셋 분할을 수행할 수 있다.

## 4. 워크스페이스(workspace)의 구성

워크스페이스는 원본 이미지와 라벨링 이력을 체계적으로 관리하기 위해 다음과 같은 디렉토리 구조로 구성된다.

```
/(workspace_name)
/---(src)
/------0000000001.jpg
/------0000000002.png
/------0000000003.jpeg
/------ ...
/---(label)
/------0000000001_C.json              # 작업+검수 완료된 파일
/------0000000002_W.json              # 작업 중인 파일
/------0000000003.json                # 작업 전 (비어있는) 파일
/------ ...
/workspace.yaml
```

- **파일명 규칙**: 원천데이터 파일명은 반드시 **9자리의 숫자(digit)** 로 규칙화된다. 워크스페이스 생성 시 원본 파일의 유효성 검사 후 일괄 변환(복사)된다.
- **라벨 파일 매핑**: 라벨링 데이터는 원천데이터의 고유 digit과 1:1 대응한다.
    - 파일명 뒤 **`_C`** (언더바+C): 검수 및 작업이 완벽히 마무리된 상태 (Completed)
    - 파일명 뒤 **`_W`** (언더바+W): 라벨링이 진행 중이거나 아직 검수가 끝나지 않은 상태 (Working)
    - **접두어 없음**: 작업을 시작하지 않은 빈 파일 상태
- 워크스페이스가 최초 생성될 때, 원천데이터 수와 일치하는 비어있는 `.json` 빈 파일이 일괄 생성된다.

### 4.1 `workspace.yaml`의 구성

해당 워크스페이스의 메타데이터를 담고 있으며, 프로그램 진입 및 설정 로드 역할을 한다.

```yaml
workspace: workspace_name
labeling_type: 2                # 1: BB, 2: OBB
names:                          # 클래스 ID 및 이름 정의
  0: bicycle
  1: car
  2: motorcycle
  3: airplane
  4: bus
image_count: 5000               # 포함된 원천데이터 총 개수
created_at: 'xxxx-xx-xx'
last_modified_at: 'xxxx-xx-xx'
```

## 5. 라벨 데이터 (`.json`) 구조

정규화로 인한 좌표 손실을 막기 위해 모든 라벨 데이터는 이미지의 오리지널 픽셀 사이즈를 기준으로 저장된다.

### 5.1 Bounding Box (타입 1) 저장 방식

일반적인 직사각형 형태의 2D 바운딩 박스를 저장한다.

```json
{
  "image_info": {
    "filename": "0000000001.jpg",
    "width": 1920,
    "height": 1080
  },
  "annotations": [
    {
      "id": "uuid-string",
      "class_id": 1,
      "bbox": [xmin, ymin, xmax, ymax]   // 절대 좌표 (정수)
    }
  ]
}
```

### 5.2 Oriented Bounding Box (타입 2) 저장 방식

회전 바운딩 박스는 중심점 좌표, 너비, 높이, 각도를 기준으로 저장된다. 이 방식은 UI에서의 크기/각도 조절 및 렌더링에 직관적이고 유리하다.

```json
{
  "image_info": {
    "filename": "0000000002.png",
    "width": 1920,
    "height": 1080
  },
  "annotations": [
    {
      "id": "uuid-string",
      "class_id": 2,
      "obb": [cx, cy, w, h, angle]
    }
  ]
}
```

- `cx`, `cy`: 회전 바운딩 박스 중심점의 절대 픽셀 좌표
- `w`, `h`: 회전 바운딩 박스의 픽셀 단위 너비와 높이
- `angle`: 지정된 기준축(x축) 대비 회전 각도 (도(Degree) 또는 라디안(Radian) 사용)

## 6. 핵심 기능 및 UI/UX 고려사항

- **Canvas 제어**: 마우스 휠 및 드래그, 단축키를 이용한 부드러운 Zoom In/Out 및 Panning 지원으로 미세한 객체 작업성 확보.
- **단축키 (Hotkeys) 최적화**: 손을 마우스에서 떼지 않고 키보드만으로 클래스 변경(Number Key), 도구 전환, 다음/이전 이미지로 이동(A/D등), 저장(Ctrl+S) 가능하게 구성.
- **대시보드 지원 (Workspace Status)**: 전체 원천데이터 수량 대비 작업 완료(`_C`), 작업 중(`_W`), 미작업 파일의 비율과 현황을 시각화.
- **히스토리 관리**: 실수 복구를 위한 Undo/Redo 기능 지원.

## 7. 내보내기 (Export) 파이프라인

작업이 끝난 검수 완료 데이터를 AI 모델이 학습할 수 있는 형태로 패키징한다.

- **내보내기 대상 플래그**: Export 시 `검수 완료만 내보내기` 플래그를 제공하며, 기본값은 활성화(ON)이다. 활성화 시 `_C.json` 상태만 포함하고, 비활성화 시 작업 중(`_W.json`) 데이터를 포함한 전체 라벨 데이터를 대상으로 내보낼 수 있다.
- **지원 포맷 구조**:
    - BB 모델용: YOLO (txt 형식, 정규화된 `cx cy w h`), COCO (json 형식)
    - OBB 모델용: YOLO-OBB, DOTA 포맷 (필요시 내보내기 단계에서 프론트의 `[cx, cy, w, h, angle]` 값을 삼각함수로 자동 계산해 모서리 4점 좌표 `x1, y1, x2, y2, x3, y3, x4, y4`로 변환하여 출력 지원)
- **데이터 증강 및 사이즈 조정**: 학습 환경에 맞춘 이미지 리사이즈 옵션 지원 (리사이즈 비율에 맞춰 라벨 좌표는 자동 재조정).
- **데이터셋 분할**: 사용자가 지정한 비율(예: Train 80%, Val 10%, Test 10%)에 맞추어 이미지를 무작위 분할하고, 올바른 폴더 및 디렉토리 구조를 자동 생성.

## 8. 전체 워크플로우 (Workflow) 요약

본 라벨링 툴은 원천데이터의 도입부터 최종 AI 학습용 데이터셋 추출까지 직관적이고 끊김 없는 파이프라인을 제공한다. 전체 프로세스는 다음 5단계로 진행된다.

### 🔄 워크플로우 다이어그램

`[1. 워크스페이스 생성]` ➔ `[2. 프로젝트 설정]` ➔ `[3. 라벨링 작업 (AI 보조)]` ➔ `[4. 검수 및 완료]` ➔ `[5. 데이터셋 내보내기]`

---

### Step 1. 워크스페이스 생성 (Workspace Initialization)

1. **데이터 가져오기**: 사용자가 원천데이터(이미지 파일들)가 담긴 폴더를 선택하거나 드래그 앤 드롭한다.
2. **데이터 규격화**: 시스템이 이미지의 유효성(확장자, 손상 여부 등)을 검증한 뒤, 파일명을 **9자리 숫자 연속 번호(예: `0000000001.jpg`)** 로 일괄 변환하여 `/(workspace)/src/` 폴더로 복사(또는 이동)한다.
3. **라벨 템플릿 생성**: 원천데이터 수량과 1:1로 매칭되는 비어있는 JSON 파일(`0000000001.json`)들을 `/(workspace)/label/` 폴더에 일괄 생성한다.

### Step 2. 프로젝트 설정 (Configuration)

1. **라벨링 타입 지정**: 과업의 목적에 따라 **BB(타입 1)** 또는 **OBB(타입 2)** 중 하나를 선택한다.
2. **클래스 정의**: 라벨링할 객체의 ID와 이름(Names)을 정의한다.
3. **초기화 완료**: 설정된 메타데이터가 `workspace.yaml`에 저장되며, 대시보드에서 전체 데이터 수량 및 진행률(0%)을 확인할 수 있게 된다.

### Step 3. 데이터 라벨링 (Labeling & AI Assistance)

1. **수동/자동 라벨링**: 작업자가 캔버스 위에서 마우스와 단축키를 이용해 박스를 그린다. (※ 향후 추가될 Auto-labeling이나 SAM 기능 호출 시, 1차적인 박스가 자동으로 캔버스에 렌더링된다.)
2. **절대 좌표 저장**: 작업 이력은 정규화되지 않은 실제 픽셀 크기 기반의 절대 좌표(`cx, cy, w, h, angle` 등)로 JSON 파일에 실시간 저장된다.
3. **상태 변경**: 작업이 진행 중이거나 임시 저장된 파일은 파일명 뒤에 작업 중임을 의미하는 태그가 붙어 **`0000000001_W.json`** 으로 변경된다.

### Step 4. 검수 및 완료 (Review & Completion)

1. **데이터 검수**: 작업자 또는 검수자가 그려진 바운딩 박스의 크기, 각도, 클래스가 정확한지 확인한다.
2. **수정 및 확정**: 필요한 경우 좌표나 각도를 미세 조정한 뒤 '작업 완료(Complete)' 처리를 한다.
3. **상태 변경**: 파일명 뒤의 태그가 검수 완료를 의미하는 **`0000000001_C.json`** 으로 최종 변경된다. 대시보드의 완료 진척도가 상승한다.

### Step 5. 데이터셋 내보내기 (Export & Dataset Generation)

1. **대상 필터링**: 기본적으로 `_C.json` (검수 완료) 상태의 데이터만 내보내기 대상으로 자동 필터링된다. 필요 시 Export 옵션의 플래그를 통해 `_W.json` 등 검수 전 데이터를 포함한 전체 라벨 데이터를 대상으로 확장할 수 있다.
2. **변환 및 리사이즈 옵션 설정**:
    - 목적에 맞는 AI 포맷(YOLO, COCO, DOTA 등)을 선택한다.
    - 필요시 이미지 해상도를 리사이즈(예: 640x640)하도록 설정한다. (라벨 좌표도 비율에 맞춰 자동 스케일링됨)
3. **데이터셋 분할 및 추출**: 지정한 Train / Validation / Test 비율(예: 8:1:1)에 맞춰 데이터가 무작위 셔플링 및 분할되며, 선택한 포맷 규격(상대 좌표 정규화 등)에 맞게 변환되어 최종 학습용 폴더 구조로 추출된다.
