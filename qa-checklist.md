# QA 체크리스트 — Phase 5 릴리스 후보

릴리스 직전에 `pnpm dev`로 띄워 수동으로 돌릴 시나리오. 자동화는 이후 Playwright E2E로 이관 (Phase 7+).
각 항목은 **Pass / Fail / N/A** 로 기록.

---

## 0. 준비물

- 이미지 **1천 장** 이상이 들어있는 디렉토리 (소형 워크스페이스 검증용)
- 가능하면 **수만 장** 디렉토리 (병렬 처리·가상 스크롤 체감용)
- BB용 / OBB용 각각 최소 1개씩 수동 라벨이 가능한 이미지

## 1. 워크스페이스 생성 (BB)

- [ ] Dashboard 시작 화면 → "새 프로젝트 생성" 다이얼로그가 열린다
- [ ] 이름 / 소스 폴더 / 저장 경로 / labeling_type=BB / 클래스 N개 입력 가능
- [ ] 생성 시 진행 상태(로딩) 표시
- [ ] 1천 장 기준으로 생성 시간 **수 초 내** (병렬화 효과 확인)
- [ ] 생성 후 `workspace.yaml`, `src/0000000001.jpg` 식 파일명, `label/0000000001.json` 빈 JSON들이 만들어진다
- [ ] 워크스페이스가 자동으로 열리고 첫 이미지가 캔버스에 표시된다

## 2. BB 라벨링 (핵심 경로)

- [ ] `B` 키 → 박스 생성 모드 진입, 툴팁 표기 `Box (B)`
- [ ] 드래그로 박스 생성, 자동으로 현재 선택된 클래스로 배정
- [ ] **Shift 드래그 중**: 정사각형으로 고정됨
- [ ] **Alt 드래그 중**: 시작점이 중심이 되어 양방향 리사이즈됨
- [ ] 박스 클릭 → RightSidebar "Labels" 리스트 하이라이트 (캔버스 → 리스트)
- [ ] 리스트 항목 클릭 → 캔버스의 해당 박스 선택 (리스트 → 캔버스)
- [ ] Shift-click으로 다중 선택, 선택 스타일이 일괄 적용됨
- [ ] `Delete` / `Backspace` → 선택 전부 삭제
- [ ] 박스 리사이즈/이동 후 autosave 인디케이터 "저장 중 → 저장됨 방금"
- [ ] `Ctrl+S` → 인디케이터가 즉시 "저장됨 방금"으로 전환

## 3. OBB 라벨링

- [ ] labeling_type=OBB 워크스페이스에서 박스 생성 시 회전 가능한 OBB로 표시
- [ ] 핸들로 회전 → angle이 라벨 JSON에 반영
- [ ] DOTA / YOLO-OBB 내보내기에서 4개 코너가 정확히 계산됨 (스모크)

## 4. Undo / Redo

- [ ] 이미지 A에서 박스 2개 추가 → `Ctrl+Z` 두 번 → 0개
- [ ] `Ctrl+Y` 두 번 → 2개 복구
- [ ] 이미지 B로 이동 → 다시 A로 복귀 → **Undo/Redo 히스토리가 유지됨**
- [ ] 이미지 B에서의 편집 이력도 B로 돌아오면 그대로 있음
- [ ] 스택이 50개를 넘기면 가장 오래된 것이 쫓겨남 (스트레스 케이스)

## 5. Copy / Paste

- [ ] 박스 N개 선택 → `Ctrl+C`
- [ ] 다른 이미지로 이동 → `Ctrl+V` → 같은 좌표·클래스로 새 UUID 박스 생성
- [ ] BB 워크스페이스에서 복사 → OBB 워크스페이스로는 **붙여넣기 되지 않아야 함** (모드 일치 필터)

## 6. 클래스 재지정

- [ ] ProjectSettingsDialog에서 클래스별 어노테이션 개수 배지가 표시
- [ ] "클래스 일괄 재지정" → A 클래스 → B 클래스, 확인 후 전체 워크스페이스의 어노테이션이 실제 전환
- [ ] 사용 중인 클래스 삭제 시 "전체 삭제 / 다른 클래스로 재할당" 선택 모달이 뜸
- [ ] 재할당 후 해당 클래스 카운트가 0이 되고, 대상 클래스 카운트가 증가

## 7. Export 프리플라이트 + 포맷

각 포맷별로 BB / OBB 각각 한 번씩:

- [ ] YOLO (BB)
- [ ] YOLO-OBB
- [ ] COCO
- [ ] DOTA

각 포맷에서:

- [ ] 프리플라이트 요약에 총 이미지·어노테이션·split 분포·클래스별 개수가 표시됨
- [ ] 범위 초과 어노테이션 있으면 경고 리스트
- [ ] resize on/off 토글
- [ ] out-of-bounds 정책 clip / skip / none 토글
- [ ] 확인 후 내보내기 → 실제 출력 디렉토리에 `train/val/test` 구조와 이미지·라벨이 생성됨
- [ ] train/val/test 각 split에 최소 1개씩 (비율이 모두 >0일 때)

## 8. 이미지 리스트 사이드패널

- [ ] 수천 장 워크스페이스에서도 스크롤이 부드러움 (가상 스크롤)
- [ ] 상태 dot (none / W / C)과 C/W 배지가 파일별로 정확
- [ ] 현재 이미지로 자동 스크롤
- [ ] 리스트 클릭 시 해당 이미지로 이동

## 9. Minimap

- [ ] 썸네일에 뷰포트 사각형이 표시됨
- [ ] Minimap 안을 클릭·드래그하면 메인 캔버스 팬이 따라감
- [ ] 줌/팬 후에도 뷰포트 사각형 위치가 맞음

## 10. 모드 전환 (Edit / Check / Preview)

- [ ] `Tab` 한 번: Edit → Check
- [ ] `Tab` 한 번 더: Check → Preview → Canvas 대신 Dashboard 표시
- [ ] `Shift+Tab`으로 역방향 순환
- [ ] Dashboard: `_C`/`_W`/미작업 진척도 카드, 누적 바, 클래스 일람, 경로·타임스탬프

## 11. 최근 워크스페이스 / 창 상태

- [ ] 워크스페이스 하나 이상 연 적 있으면 시작 화면 드롭다운에 노출
- [ ] 존재하지 않는 경로는 자동 필터링
- [ ] 앱 종료 후 재실행 시: 창 크기·위치·maximized 상태 복원
- [ ] 다크 모드 토글이 즉시 반영되고 재실행 후 유지

## 12. 저장 인디케이터 / 자동 저장

- [ ] 박스 편집 → Footer "저장 중…" → "저장됨 방금"
- [ ] 500ms 이내 연속 편집은 디바운스되어 한 번만 저장
- [ ] 1분 넘게 방치 시 "N분 전"
- [ ] 오류 주입 (예: `label/` 디렉토리 쓰기 권한 제거) → "저장 실패" 상태 표시

## 13. 단축키 툴팁

- [ ] 모든 툴바 버튼·Navigation 버튼에 호버 시 설명 + 단축키 표시
- [ ] 표시된 단축키가 실제로 동작함

## 14. 회귀 위험 포인트

- [ ] Undo/Redo 스택 50 상한 후의 동작이 이상하지 않음
- [ ] 동일 이미지에 BB 100+ 개 올려도 캔버스 반응성 유지
- [ ] Export 시 범위 초과 어노테이션이 있는 워크스페이스에서 skip / clip / none 모두 기대대로 동작
- [ ] 한글/공백/특수문자가 포함된 워크스페이스 이름으로 Export 경로가 올바르게 sanitize됨
